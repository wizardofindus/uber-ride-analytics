# Ride-Sharing (Uber) Analytics Dashboard

**SQL + Python + MySQL + Power BI**

---

##  Overview

This project analyzes ride-sharing (Uber/taxi) trip data to uncover **demand patterns, pricing behavior, and operational insights**.
It demonstrates an end-to-end workflow using **SQL for data processing, Python for analysis & feature engineering, and Power BI for interactive visualization**.

---

##  Objectives

* Analyze **trip demand by time and location**
* Identify **peak hours and high-demand zones**
* Understand **fare vs distance relationship**



---

##  Tech Stack

* **SQL (MySQL)** → Data storage, cleaning, analytics queries
* **Python (Pandas, NumPy, Seaborn, Matplotlib)** → EDA & feature engineering
* **Power BI** → Dashboard & visualization
* **GitHub** → Version control and project showcase


##  Dataset

* Ride-sharing / taxi trip dataset (NYC / Uber-style data)
* Includes:

  * Pickup & dropoff datetime
  * Pickup & dropoff coordinates
  * Fare amount
    

###  Visual Insights

*  Monthly demand trends
*  Hourly trip distribution (peak analysis)
*  Day-of-week demand patterns
*  Fare vs distance relationship
*  Pickup location heatmap (geospatial analysis)
*  Peak vs non-peak revenue comparison
